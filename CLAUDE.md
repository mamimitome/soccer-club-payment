# サッカークラブ月謝管理システム - Claude 開発ルール

このファイルはClaudeがこのプロジェクトで作業する際に従うべきルールを定義します。

---

## コーディングルール

### コメント
- **すべてのコードに日本語で詳細なコメントを入れること**
- 対象読者：エンジニア未経験者・SES研修中の方が読んでも理解できるレベル
- コメントに含めるべき内容：
  - そのコードが「何をしているか」だけでなく「なぜそうしているか」
  - 専門用語が出てきたら括弧内で説明を添える（例：「外部キー（他のテーブルと紐付けるための仕組み）」）
  - 複数の選択肢がある場合は、なぜその方法を選んだか理由を書く

### コミットメッセージ
- **コミットメッセージは必ず日本語で書くこと**
- 形式：`種別: 変更内容の要約`
- 種別の例：`追加`, `修正`, `削除`, `リファクタリング`, `設定`
- 例：`追加: paymentsテーブルのマイグレーションを作成`

### 破壊的な変更
- **ファイルの削除・上書き・データベースのリセットなど取り消しのきかない操作は、必ず日本語で説明してから実行すること**
- 説明に含める内容：何を削除・変更するか、なぜ必要か、影響範囲
- ユーザーの確認を得てから実行する

## ビジネスルール

### 料金設定
| 会員種別 | 支払い方法 | 金額 |
|---------|-----------|------|
| 正会員（member） | 月謝・クレジットカード自動引き落とし | **9,000円／月** |
| ビジター（visitor） | 都度払い・クレジットカード | **2,500円／回** |

### ユーザーロール
| ロール名 | 説明 | 権限 |
|---------|------|------|
| `admin` | 管理者（クラブスタッフ） | すべての操作が可能 |
| `member` | 正会員 | 自分の情報・決済履歴の閲覧 |
| `visitor` | ビジター | 都度払い決済・履歴の閲覧 |

## 技術スタック

| 分類 | 技術 | 用途 |
|------|------|------|
| バックエンド | Laravel 12 | メインフレームワーク |
| 認証 | Laravel Sanctum | ログイン・セッション管理 |
| フロントエンド | Blade テンプレート | HTMLテンプレートエンジン |
| CSS | Tailwind CSS v4 | スタイリング |
| データベース | MySQL | データ永続化 |
| 決済 | Stripe | クレジットカード決済処理 |

### 認証
- 認証には **Laravel Sanctum** を使用すること（JWT・Passport は使用しない）
- セッションベース認証（Webアプリ用）を使用する

### データベース
- データベース名：`soccer_club_payment`
- 文字コード：`utf8mb4`（日本語・絵文字に対応）
- 照合順序：`utf8mb4_unicode_ci`
